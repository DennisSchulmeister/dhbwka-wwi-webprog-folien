Was als einfache Auszeichnungssprache für wissenschaftliche Publikationen am CERN anfing ist heute richtig mächtig geworden, so dass die Webseiten von heute nichts mehr mit den einfachen Textdokumenten von damals zu tun haben. Natürlich weißt du aus deiner Erfahrung bereits, wie unterschiedlich die Webseiten von heute aussehen können und welch unbegrenzten Möglichkeiten ein Browser heute bietet. Die nebenstehenden Screenshots sollen dir trotzdem nochmal einen kleinen Eindruck vermitteln, was man mit HTML heute alles anstellen kann.

Einfache Textseiten
Das Herzstück von HTML bleiben natürlich nach wie vor die Anweisungen zur Strukturierung einfacher Textdokumente. Überschriften, Absätze, Abbildungen, Tabellen, Verweise und so weiter. HTML kennt für die meisten Dokumenteninhalte einen passenden Befehl.
Komplexe Webseiten
Moderne Webseiten sehen natürlich längst nicht mehr so einfach wie früher aus. Oftmals steht auch gar nicht mehr ein langer Fließtext im Vordergrund sondern eher die visuell ansprechende Aufbereitung einzelner Informationen und multimedialer Inhalte.
Browser Games
Seit es Computer gibt, gibt es Computerspiele. Wirklich leistungshungrige Spiele laufen natürlich weiterhin außerhalb des Browsers. Da aber inzwischen selbst 3D-Grafiken im Browser berechnet werden können, lassen sich viele Spiele der vergangenen Jahre inzwischen online spielen. Siehe zum Beispiel The WebGL Bookcase für ein einfaches aber beeindruckendes Beispiel.
Browser Apps
Dank HTML5 und modernen JavaScript APIs handelt es sich beim Browser längst nicht mehr um ein einfaches HTML-Anzeigeprogramm sondern um eine vollständige Laufzeitumgebung für grafische Anwendungen. Immer mehr Anwendungen werden daher in Form von SaaS-Cloudangeboten direkt im Browser genutzt. Dropbox, Google Documents oder Microsoft Live sind nur ein paar der bekanntesten Beispiele, aber auch On-Premise-Anwendungen, die nicht in der Cloud laufen, haben oft eine webbasierte Benutzeroberfläche.
Smart TV Apps
Dieses Beispiel mag auf den ersten Blick etwas überraschen. Denn tatsächlich galten Fernsehr die längste Zeit als nicht-programmierbare Geräte, die einfach nur das tun, was sie eben tun (vgl. Eingebettetes System). Neuere Geräte erlauben aber durchaus die Installation eigener Apps und inzwischen hat sich HTML hier als UI-Technologie gegenüber anderen Programmiersprachen durchgesetzt, da dies einen besonders leichten Ein- und Umstieg ermöglicht.
Mobile Apps
Bei mobilen Apps für Smartwatch, Smartphone und Tablet gibt es im Prinzip zwei Arten: Die Anwendungen, die mit dem SDK des jeweiligen Herstellers entwickelt wurden und daher nur auf einer bestimmten Geräteklasse laufen (z.B. Android oder iOS) und die, die mit HTML realisiert wurden und daher auf vielen verschiedenen Geräten lauffähig sind. Schätzungsweise handelt es sich dabei um rund die Hälfte aller mobilen Anwendungen. Werkzeuge wie Apache Cordova oder sein kommerzieller Ableger Phonegap erlauben es, komplette Apps nur mit HTML, CSS und JavaScript zu entwickeln.

Du siehst, ohne zu übertreiben kann man sagen, dass HTML inzwischen zu einer universellen UI-Technologie geworden ist. Egal ob groß oder klein, HTML ist auf jedem Gerät daheim und eignet sich zur Gestaltung fast jeder beliebigen Benutzeroberfläche. Doch HTML alleine reicht hierfür noch nicht ganz aus. Auf der nächsten Folie siehst du, welche Sprachen und Dateiformate zusammen das „Ökosystem” rund um HTML bilden, das die Erstellung so unterschiedlicher Oberflächen erst ermöglicht.

Sprachen und Formate rund um HTML

HTML (HyperText Markup Language)

  • Auszeichnungssprache zur Definition der darzustellenden Inhalte
  • Beschreibt nur den Inhalt und seine Bedeutung, nicht aber das Aussehen
  • Überschriften, Absätze, Aufzählungen, Tabellen, Bilder, …

CSS (Cascading StyleSheets)

  • Sprache zur Definition der Layout- und Darstellungsoptionen einer HTML-Seite
  • Bezieht sich auf die Inhalte eines HTML-Dokuments und legt ihre Darstellung fest
  • Farben, Hintergründe, Umrandungen, Positionierung, Transparenz, Schatten, …

SVG (Scalable Vector Graphics)

  • XML-basiertes Grafikformat für skalierbare Vektorgrafiken
  • Kann mit CSS und JavaScript umfangreich zur Laufzeit angepasst werden
  • Ist daher gut zur Definition interaktiver, grafischer Elemente geeignet
  • Rechtecke, Kreise, Linien, Kurven, Flächen, Füllungen, …

JavaScript

  • Programmiersprache, die innerhalb des Browsers ausgeführt wird
  • Ermöglicht es, auf Ereignisse wie Mausklicks zu reagieren
  • Ermöglicht die nachträgliche Veränderung der sichtbaren Seiteninhalte
  • Bietet ein umfangreiches API zur App-Entwicklung

Dass die Beispiele auf der vorherigen Folie nur auf HTML basieren, ist so nicht richtig. Denn streng genommen handelt es sich bei HTML nur um den Teil zur Strukturierung der Inhalte. Zwar werden der grundlegende Aufbau einer Oberfläche sowie alle darin enthaltenen Informationen in HTML definiert, ohne ein paar Freunde geht es dann allerdings doch nicht. 😎

Jede HTML-Webanwendung besteht daher aus einer Kombination der folgenden Programmiersprachen, die Hand in Hand zusammenspielen:

HTML (HyperText Markup Language)
Mit HTML werden die darzustellenden Inhalte definiert und die „Bereiche” einer Seite festgelegt. Hierfür werden in einer HTML-Datei alle Textinhalte wie Überschriften, Absätze oder Abbildungen festgelegt und mit Namen versehen, um sie später zum Beispiel zu einem Kopfbereich oder einer Side Bar gestalten zu können.
CSS (Cascading StyleSheets)

Das Aussehen der HTML-Inhalte wird in einem sogenannten Stylesheet definiert. Denn HTML selbst beinhaltet keine Anweisungen hierfür. Stattdessen sollen Inhalt und Aussehen getrennt geregelt werden, um eine bessere Wiederverwendbarkeit des Layouts zu erzielen.¹ Hier kommen dann die Namen ins Spiel, die man den einzelnen Elementen einer HTML-Datei gibt. Denn darüber lassen sie sich im Stylesheet ansprechen und gestalten.

¹ Beispielsweise um zwei HTML-Dateien dasselbe Layout zu geben. Oder um an Weihnachten seiner Webseite ein anderes Layout zu geben, ohne alle HTMLs anpassen zu müssen. 🎅 🎄

SVG (Scalable Vector Graphics)
SVG ist eines der verschiedenen Grafikformate im Web. Das Besondere daran ist, dass es sich um ein Vektorgrafikformat (statt einem Rastergrafikformat) handelt, das mit CSS und JavaScript komplett skriptbar ist. Somit lassen sich mit SVG zum Beispiel interaktive Schaubilder und andere grafische Elemente erstellen.
JavaScript
Die Schaltzentrale, die allen Webanwendungen ihre clientseitige Logik gibt, ist JavaScript. Einzelheiten dazu wirst du später noch erfahren. Hier reicht es erst mal zu wissen, dass es sich um eine vollwertige Programmiersprache handelt, die eingebettet in einer Webseite im Browser läuft und alle Inhalte der Seite ändern kann. Zusammen mit ihrer umfangreichen Programmierschnittstelle (API) zur App-Entwicklung lassen sich damit fast beliebige Anwendungen entwickeln.

Exkurs: Bilder für das Web aufbereiten

Bilder sind für viele Webseiten unverzichtbar. Egal ob als Hintergrundbild, als lustiges Katzenbild oder als echt seriöse Abbildung: Bilder gehören einfach dazu. Falsch eingesetzt können sie aber auch einer der Hauptgründe für lange Ladezeiten und frustrierte Besucher sein. Denn im Web gilt, dass eine Seite so schnell wie nur irgend möglich laden sollte, damit der Besucher nicht das Interesse verliert und einfach weiter surft. Dabei solltest du auch beachten, dass viele Besucher inzwischen viel mit dem Smartphone unterwegs sind und das Problem da noch viel größer ist. Denn mobile Geräte haben oft nur eine schlechte Internetverbindung und lange Ladezeiten sind der Tod eines jeden Akkus. 😒

Aus diesem Grund gilt für Grafiken im Web im wahrsten Sinne „weniger ist mehr“. Sowohl mit Blick auf die Anzahl als auch ihre Dateigröße. Welche unnötigen Bilder du auf deiner Seite eliminieren kannst, können wir dir natürlich nicht sagen. 😉 Nachfolgend findest du aber ein paar Tipps, wie du wenigstens die Dateigröße klein halten kannst.

Wähle das richtige Dateiformat

Noch bevor du eine Grafik erstellst solltest du dir Gedanken darüber machen was sie darstellen soll und welches Dateiformat dafür in Frage kommt. Denn mit dieser einfachen Entscheidung kannst du bereits die größten Einsparungen erzielen oder eben auch den meisten Platz verschwenden. Zwischen folgenden Formaten kannst du dabei wählen:

PNG (Portable Network Graphics)

PNG ist sozusagen das Standardformat für Abbildungen im Web und ein moderner Nachfolger für das alte GIF-Format. Wie bei den meisten Formaten handelt es sich um ein sogenanntes Rastergrafikformat, was bedeutet, dass die Bilder Pixel für Pixel gespeichert werden. Vorteile des Formats sind:

  • Verlustfreie Komprimierung der Bilder¹
  • Untersützung jeder beliebigen Farbtiefe ohne Farbrasterung²
  • Unterstützung eines Alphakanals für halbdurchsichtige Pixel³

Jedoch ist das Format nicht für alle Arten von Bildern geeignet. Insbesondere fotoähnliche Bilder mit vielen Farben führen zu sehr großen Dateien. Am besten eignet es sich daher für Screenshots, bei denen es auf die Lesbarkeit der Schrift ankommt sowie für Bilder mit wenigen Farben oder mit halbdurchsichtigem Hintergrund.

¹ Durch die Komprimierung gehen keine Bildinformationen verloren.

² Es wird keine Farbrasterung benötigt, um die Anzahl der Farben zu reduzieren. Andere Formate arbeiten hier mit einer sogenannten Farbpalette und können nur eine bestimmte Anzahl Farben gleichzeitig darstellen, meist nur 256 Stück.

³ Zu jedem Pixel wird zusätzlich noch ein Prozentwert gespeichert, wie durchsichtig oder deckend er dargestellt wird.

GIF (Graphics Interchange Format)
Das GIF-Format wurde bereits 1988 noch vor dem Web erfunden. Lange Zeit galt es als das Standardformat für den Austausch von Bildern zwischen Computerbenutzern. Im Vergleich zu PNG hat es aber die Nachteile, dass es nur 256 Farben speichern kann und dass keine halbtransparenten Pixel unterstützt werden. GIF-Grafiken mit durchsichtigem Hintergrund haben daher so gut wie immer unschöne Kanten. Jedoch kann man damit einfache Animationen erstellen, was mit PNG nicht möglich ist. Beispiel für ein animiertes GIF. Bildnachweis: Wikipedia
JPEG (Joint Photographic Experts Group)
Dies ist das bekannte Standardformat für komprimierte Fotos, das von jeder beliebigen Digitalkamera erzeugt wird. Im Vergleich zu PNG sind die Dateien sehr viel kleiner, die Komprimierung ist dafür aber verlustbehaftet. Durchsichtige Hintergründe sind ebenfalls nicht möglich. JPEG sollte daher das Standardformat sein, wenn du Fotos oder farbreiche Screenshots einbinden willst.

Bei Fotos mit weichen Kanten fallen die Kompressionsverluste oft nicht auf, bei harten Kanten und bei Schriftzeichen dafür umso mehr. Zu sehr komprimierte Bilder beinhalten gut sichtbare „Blöckchen” mit verzerrten Farben und in der Nähe von Buchstaben und Wörtern finden sich häufig rauschende Streupixel.

SVG (Scalable Vector Graphics)

SVG ist ein XML-basiertes Format für skalierbare Vektorgrafiken (sagt ja schon der Name 🙃). Anstelle einzelner Pixel werden nur Anweisungen gespeichert, anhand derer sich eine Zeichnung rekonstruieren lässt. SVG-Grafiken sehen daher in jeder Größe und Auflösung gestochen scharf aus. Das Format eignet sich daher gut für Schaubilder und andere technische Zeichnungen, richtig angewendet können aber auch ansprechende Zierelemente damit gestaltet werden. Des weiteren hat es noch folgende Vorteile:

  • SVG-Befehle können über das <svg>-Tag direkt in HTML eingebunden werden.
  • Einzelne Elemente einer SVG-Grafik können mit CSS formatiert werden. Man kann damit z.B. die Farbe oder Art einer Linie nachträglich zur Laufzeit ändern.
  • Zusätzlich können die Elemente einer SVG-Grafik auch mit JavaScript angesprochen und manipuliert werden. Man kann also ohne weiteres auch interaktive Grafiken erstellen, die zum Beispiel auf Maus- oder Touchgesten reagieren.

So klein wie möglich, so groß wie nötig

Dies ist eine sehr wichtige Regel für Rastergrafiken im Web. Moderne Kameras erzeugen oft extrem große Bilder mit einer sehr hohen Auflösung. Damit eignen sich die Bilder gut für den Druck oder eine Vollbildanzeige auf großen Bildschirmen. Für Webseiten sind sie aber viel zu groß! Bevor du also ein Foto in deine Seite einbindest, solltest du es unbedingt verkleinern. Für die genaue Größe gibt es keine magischen Werte, die immer funktionieren. Letztlich musst du einfach versuchen, einen Mittelweg zwischen kleiner Dateigröße und guter Darstellung zu finden. Folgende Zahlen können dir dabei als Hilfe dienen:

  • Kleine Bilder: ca. 500 Pixel hoch oder breit, ca. 50 kB
  • Mittlere Bilder: ca. 800 bis 1000 Pixel hoch oder breit, ca. 120 kB
  • Große Bilder: ca. 1000 bis 1500 Pixel hoch oder breit, ca. 250 kB

Diese Werte sind sicher sehr konservativ geschätzt und du wirst sie nicht immer einhalten können. Und je mehr die Bildschirmauflösungen und Internetbandbreiten im Laufe der Zeit zunehmen werden, desto größer können die Zahlen werden. Dennoch solltest du wirklich um jeden Preis versuchen, jedes unnötige kB einzusparen. Denn es gibt immer einen Anwender, der gerade im tiefsten Urwald (oder im kanadischen Hinterland!) sitzt und über eine instabile und mega-langsame Funkverbindung surft, die ständig zusammen bricht (haben wir in Kanada wirklich erlebt). Deshalb solltest du die Anzahl der Pixel immer klein halten, auch wenn die meisten Bildschirme und Mobilgeräte inzwischen Full HD und mehr darstellen können.

Ein weiterer Trick aus dieser Rubrik ist die Verwendung unterschiedlicher Bilder für unterschiedliche Endgeräte. Dies kannst du entweder mit CSS Media Queries oder mit dem (noch nicht von allen Browsern unterstützten) <picture>-Tag erreichen. Kleinen Geräten mit in der Regel wenig Bandbreite kannst du damit kleinere Bilder senden, als größeren Geräten mit großen Bildschirmen und meistens üppigem Internetanschluss.

Optimiere die Exportparameter

Was uns nun endlich zum Screenshot auf der Folie bringt. Gute Grafikprogramme wie The Gimp erlauben es, das Dateiformat beim Speichern bzw. Exportieren eines Bildes nochmal zu tunen, um kostbare kB zu sparen. Teilweise kann sich die Größe einer Datei dadurch um die Hälfte reduzieren. Der Screenshot zeigt daher eine gute Voreinstellung, die du als Grundlage für eigene JPEG-Bilder verwenden kannst:

  • Show preview in image window: Wenn dieses Häkchen gesetzt ist, siehst du im Exportfenster eine Berechnung der erzielten Dateigröße. Gleichzeitig kannst du das Ergebnis im Hauptfenster von GIMP kontrollieren, da dort eine exakte Vorschau angezeigt wird, wie die komprimierte Grafik später aussehen wird. Achte deshalb besonders auf Kompressionsartefakte und entscheide, ob du mit ihnen leben kannst (ohne geht nicht) oder ob du doch lieber ein paar mehr Bytes spendieren willst.
  • Quality: Das ist der wichtigste Parameter. Je niedriger der Wert, desto kleiner die Datei und desto schlechter die Bildqualität. Je nach Grafik kann man auch mal bis 50% oder tiefer gehen, meistens benötigt man dann aber doch so um die 60% bis 80%, damit es noch gut aussieht.
  • Subsampling: 4:2:2 Hierdurch wird die Farbtiefe reduziert, was meistens wenn überhaupt nur auf den zweiten Blick auffällt. Jedoch kann damit in der Regel rund ein Drittel der Dateigröße eingespart werden.
  • Optimize und Progressive: Drücken die Dateigröße nochmals um ein paar kB. Die Auswirkung ist nicht so groß, aber es kommt ja bekanntlich auf jedes Byte an.
  • Save EXIF-Data, Save thumbnail, Save XMP data, Comment: Das sind alles unnötige Zusatzdaten, die der Browser nicht darstellen kann, die aber Platz belegen. Diese Häkchen solltest du für Webgrafiken immer ausschalten.
