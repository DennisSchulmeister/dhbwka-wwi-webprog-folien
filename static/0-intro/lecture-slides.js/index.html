Tastatur- und Touchbedienung

Natürlich kannst du die einfach die Befehle in der Toolbar oben anklicken, um dich durch die Seiten zu hangeln. Für die meisten Dinge kannst du aber auch einfach die Tastatur oder die bekannten Touchgesten nutzen.

Tastenkürzel

Pfeil links      Gehe zur vorherigen Folie
Pfeil reichts      Gehe zur nächsten Folie
Leertaste      Gehe zur nächsten Folie
Enter      Gehe zur nächsten Folie
P      Präsentationsmodus umschalten
O      Übersicht umschalten
ESC      Übersicht umschalten
B      Anzeige ausschalten (schwarz)
W      Anzeige ausschalten machen (weiß)

Touchgesten

Links wischen      Gehe zur nächsten Folie
Rechts wischen      Gehe zur vorherigen Folie
Doppelt tippen      Präsentationsmodus umschalten
Lang drücken      Übersicht umschalten

Eigene Folien erstellen

Im Prinzip handelt es sich bei den Folien um einfache HTML-Dateie. Zum Starten benötigst du …

  • Das Vorlageprojekt für Folien
  • Eigene Inhalte in der Datei static/index.html
  • Den richtigen Befehl, um die finalen Dateien zu bauen

Mit folgenden Befehlen wird der Quellcode der Folien gebaut:

  • Einmalig am Anfang: yarn install
  • Während der Entwicklung: npm run devserver
  • Zum Hochladen auf einen Server: npm run build

Der zweite Befehl führt einen lokalen Webserver aus, der auf Port 8080 lauscht. Öffne einfach http://localhost:8080, um die Seite zu sehen.

Die Einstieg in lecture-slides.js ist ganz einfach, zumdindest wenn du dich ein wenig mit HTML auskennst und du keine Angst davor hast, ab und zu die Kommandozeile zu benutzen. Nimm diese Präsentation zum Beispiel. Sie besteht gerade mal aus einer HTML-Datei und kann mit nur einem Befehl erzeugt werden.

Aber alles der Reihe nach. Damit du deine eigenen Folien bauen kannst, brauchst du erst ein paar Dinge. Und zwar hauptsächlich eine Kopie des Vorlageprojekts für Folien.

Herunterladen des Vorlageprojekts

Rufe einfach den Link auf das Vorlageprojekt für Folien auf, damit zu Github gelangst. Dort musst du dann auf Clone or download klicken:

Clone or download » Download ZIP

Die ZIP-Datei kannst du dann einfach irgendwo auf deinem Rechner extrahieren. Das reicht dann schon.

Deine erste eigene Folie

Nun ist es natürlich an der Zeit, deine Folien zu schreiben. Zum Glück handelt es sich dabei nur um einfache HTML-Dateien innerhalb des static-Verzeichnisses. Öffne daher einfach die Datei static/index.html im Quellcode-Editor deines Vertrauens und schaue dir an, wie sie aufgebaut ist.

Nachdem du dich ein wenig umgeschaut hast, kannst du direkt anfangen drauf los zu tippen. Lege einfach für jede Folie ein <section> an und achte darauf, dass den richtigen Inhalt hat:

  • <h1>: Überschrift der Folie
  • <article>: Die Folie selbst
  • <aside>: Erklärender Text am Seitenrand

Jedes dieser Elemente ist optional (kann also weggelassen werden). Sind sie alle vorhanden, sollte das Ergebnis jedoch so aussehen:

<section>
     <h1>Titel der Folie</h1>
     <article>Inhalt der Folie</article>
     <aside>Erklärungen am Rand</aside>
</section>

Die Präsentation bauen und laufen lassen

Nun kannst du ein Terminalfenster öffnen und das Verzeichnis mit den Quellcdatein der Folien aufrufen. DOrt angekommen, rufe einfach einen der folgenden Befehle auf:

  • Einmalig am Anfang: yarn install
  • Während der Entwicklung: npm run devserver
  • Zum Hochladen auf einen Server: npm run build

Der erste Befehl lädt die ganzen Abhängigkeiten herunter, die zum Bauen der Folien benötigt werden. Ihn musst du daher nur in jedem Projekt einmalig ausführen. Die Abhängigkeiten werden dann im neuen Unterverzeichnis node_modules gespeichert.

Der zweite Befehl führt einen lokalen Webserver aus, der auf Port 8080 lauscht. Öffne daher http://localhost:8080, um die Folien zu sehen.

Sobald du fertig bist, führe einfach den dritten Befehl aus. Er baut die fertige Version der Präsentation und legt sie im build-Verzeichnis ab. Die sich dort befindenden Dateien kannst du einfach auf irgend einen Webserver packen, um die Folien nutzen zu können. Dafür brauchst du nicht mal einen großen Server, selbst der kleinste Webserver überhaupt reicht schon aus. Oder du öffnest die HTML-Dateien einfach nur so im Browser, wenn du die Folien nur für dich erstellt hast.

Hinweis: Falls deine Folien nach dem Upload auf einen Server nicht mehr funktionieren, musst du sehr warscheinlich enie kleine Anpassung in der Datei package.json vornehmen. Und zwar immer dann, wenn die URL deiner Folien nicht zum Beispiel so lautet: http://www.example.com/ Sondern eher: http://www.example.com/slides/. In diesem Fall suche in der package.json nach folgender Zeile:

"public_url": "",

Ändere sie einfach wie folgt ab und versuche nochmal, die Präsentation zu bauen. Nun sollte alles in Ordnung sein.

"public_url": "/slides/",

Übersetzung in andere Sprachen

Standardmäßig sind die mit lecture-slide.js (so heißt diese Anwendung) erstellten Präsentationen auf englisch. Zwar kannst du deine Folien in jeder Sprache schreiben, wie du willst, die Befehle und Meldungen sind aber englisch. Das lässt sich aber ganz leicht ändern.

  1. Bearbeite die Datei index.js
  2. Übersetze alle mit label… beginnenden Werte

Beispiel für die deutsche Übersetzung:

import SlideshowPlayer from "lecture-slides.js";

window.addEventListener("load", () => {
    let player = new SlideshowPlayer({
        labelPrev: "Zurück",
        labelNext: "Weiter",
        labelGoTo: "Gehe zu",
        labelOverview: "Übersicht",
        labelPresentationMode: "Präsentationsmodus",
        labelSlide: "Folie",
        mode: "slideshow",
    });

    player.start();
});
