Das Modul „Entwicklung verteilter Systeme” besteht aus den beiden Vorlesungen „Webprogrammierung” im dritten Semester und „Verteilte Systeme” im vierten Semester, die du dir als Fortführung der Programmierkurse aus den Semestern davor vorstellen kannst. Bisher hast du in den Vorlesungen lauter Grundlagen gelernt, so dass du jetzt schon ein wenig programmieren kannst. Ab dem dritten Semester schauen wir uns daher immer mehr fortgeschrittene Themen an, um dir einen richtigen Einstieg in die Praxis zu geben.

Auf der Folie siehst du eine Übersicht der wichtigsten Themen, die wir in „Entwicklung verteilter Systeme” behandeln werden. Schauen wir uns diese mal etwas genauer an:

3. Semester: Webprogrammierung

Das World Wide Web ist die größte jemals von Menschen geschaffene verteilte Anwendung. Angefangen als einfaches Werkzeug zur Publikation wissenschaftlicher Arbeiten auf einem eigenen Server, hat es sich inzwischen zu einer ubiquitären Anwendungsplattform für grafische Anwendungen aller Art entwickelt. Wir lernen im dritten Semester daher das Web im Detail kennen und erfahren, durch welche Techniken es unter der Haube angetrieben wird. Diese schauen wir uns danach sehr ausführlich an, um am Ende des Semesters sowohl einfache Webseiten als auch ausgewachsene, mobilfreundliche Anwendungen programmieren zu können.

4. Semester: Verteilte Systeme

Über „Verteilte Systeme” möchten wir hier noch nicht so viele Worte verlieren. Wie du an den Stichpunkten aber sehen kannst, werden wir direkt dort anknüpfen, wo wir in Webprogrammierung aufhören. Nach einer kurzen Übersicht über die verschiedenen Möglichkeiten zur Gestaltung und Implementierung verteilter Anwendungen werden wir unser Wissen über Webentwicklung vervollständigen, indem wir unsere aus dem vorherigen Semester bekannten „Single Page Apps” mit einem ordentlichen REST-Backend versehen. Dieses sprechen wir dann mit so genannten AJAX-Aufrufen an, wenn eine Aktion auf dem Server ausgelöst oder neue Daten abgerufen werden sollen. Dies bildet dann die Grundlage für das eigentliche Hauptthema der Vorlesung: Die Entwicklung vollwertiger über das Web aufrufbarer Cloudanwendungen.

Für heute soll das erst einmal genügen. Im vierten Semester kommen wir nochmal darauf zurück. 😎

  • Geschrieben in HTML und JavaScript (basierend auf der Chrome Webengine)
  • Open-Source von den Machern von GitHub

Versionsverwaltung: Git
  • Verteilte Versionsverwaltung zur gemeinsamen Arbeit im Projekt
  • Zeichnet eine komplette Historie aller Änderungen an unseren Quellcodes auf
  • Ermöglicht es, beliebig vielen Personen zeitgleich an denselben Quellcodes zu arbeiten
  • Bietet zusammen mit GitHub eine komfortable Möglichkeit, Quellcode zu teilen und zu bearbeiten
  • Macht das Leben auch so in allen Belangen einfacher 👍
  • Open-Source von den Machern von Linux 🐧

Allzwecksprache: Python
  • Ganz allgemein eine nützliche Allzwekcsprache für viele Problemstellungen
  • Neben der serverseitigen Webentwicklung insb. im wissenschaftlichen Umfeld weit verbreitet
  • Beinhaltet einen kleinen Testwebserver, den wir zur Entwicklung gut gebrauchen können
  • Wird aber hauptsächlich für einen Exkurs über statische Webseitengeneratoren in der Vorlesung benötigt

Serversprache und Werkzeugsammlung: node.js
  • Beinhaltet den Node Package Manager zur einfachen Installation aller für ein Projekt benötigten Bibliotheken
  • Liefert uns alle Build- und Testwerkzeuge, die wir während der Entwicklung benötigen
  • Beinhaltet eine komplette JavaScript-Laufzeitumgebung für die serverseitige Programmierung

Atom Editor von Github

Atom ist einer der besten Editoren für Webentwickler und mittelgroße Programmierprojekte. Zwar hat er aufgrund von Visual Studio Code in den letzten Jahren etwas an Popularität eingebüßt. Im Vergleich dazu ist Atom aber wesentlich aufgeräumter und einfacher zu bedienen.

https://atom.io/

Git

Kaum ein professionelles Softwareprojekt kommt heutzutage ohne eine gute Versionsverwaltung aus. Historisch gesehen gibt es viele gute Werkzeuge hierfür, das mit Abstand beliebteste ist jedoch Git. Git ermöglicht es dir, nicht nur sämtliche Änderungen an deinem Quellcode sauber zu dokumentieren und jederzeit nachzuvollziehen, es bietet auch hervorragende Möglichkeiten zum Teamwork. Genau das richtige für unsere Vorlesung. 😃

https://git-scm.com/

Python

Python ist nicht nur bei Anfängern sehr beliebt, sondern auch bei Webentwicklern und Wissenschaftlern. Webentwickler arbeiten meistens mit einem Framework wie Django. Wissenschaftler führen umfangreiche Datenanalysen und KI-Projekte damit durch und IoT-Entwickler entwickeln selbst komplexe, verteilte Anwendungen mit Python. In der Vorlesung nutzen wir es aber hauptsächlich für einen kleinen Exkurs zum Thema statische Webseitengeneratoren.

www.python.org/

node.js und npm

Leider werden wir uns nicht sehr ausführlich mit node.js beschäftigen, auch wenn das wirklich lohnenswert wäre. Gegen Ende der Vorlesung wollen wir uns jedoch ein wenig damit beschäftigen und da macht es natürlich Sinn, sich auch die Serverseite unter JavaScript anzuschauen. node.js ist hier eine stabile und weit verbreitete Laufzeitumgebung für JavaScript außerhalb des Browsers. Vor- und Nachteile einer Klausur

Die gute Nachricht immer zuerst: Wir schreiben keine Klausur. 🎉 Wobei, so besonders ist das inzwischen ja gar nicht mehr. Waren vor ein paar Jahren noch die meisten Prüfungsleistungen schriftliche Prüfungen auf Papier, wurden moderne Prüfungsformen inzwischen ja in einem Großteil der Vorlesungen verankert. Für den Workload im Selbststudium ist das nicht immer gut, aber wenigstens machen wir kein „Programmieren auf Papier” mehr, so wie es bei den Klausuren früher immer der Fall war

Im Gegenzug sind die Prüfungsaufgaben natürlich anspruchsvoller als in einer Klausur, da ja wesentlich mehr Zeit zur Verfügung steht (nämlich das komplette Semester) und da sämtliche Hilfsmittel zur Verfügung stehen. Auf diese Weise wirst du dich aber auch wesentlich intensiver mit den Vorlesungsinhalten beschäftigen und diese dadurch viel mehr verinnerlichen.

Inhalt des Prüfungsportfolios

Wie in jeder Vorlesung beinhaltet des Prüfungsportfolio eine Reihe von Formalia (insb. Einleitung und Reflexion) sowie eine Reihe von Prüfungsartefakten, die deinen individuellen Lernweg dokumentieren sollen. Zu jedem Semester wird es daher Lernkontrollen online geben, die als Prüfungsleistung rechtzeitig bearbeitet werden müssen. Ebenso liegt für jedes Semester ein semesterbegleitendes Aufgabenblatt vor. Dieses muss unter dem Semester zu bestimmten Stichpunkten bearbeitet werden und wird am Semesterenede dann ebenfalls als Prüfungsartefakt eingereicht.

Berechnung der Note

Die Benotung funktioniert eigentlich wie in einer Klausur. Jedes Artefakt ist mit einer maximale Punktzahl versehen, die durch seine Abgabe erreicht werden kann. Am Ende werden einfach alle Punkte zusammengezählt, ihr prpzentualer Anteil zur Maximalpunktzahl ermittelt und diese anhand der Notenliste in eine Note überführt.
