In diesem Foliensatz wollen wir uns mehr mit dem Zusammenspiel von HTML und JavaScript beschäftigen. Aus diesem Grund siehst du hier nochmal zwei Beispiele dafür, wie JavaScript in eine HTML-Datei eingebunden werden kann.

Wie du siehst, ist es nicht anders, als mit CSS. Auch da hast du die Wahl, ob du den Code direkt in die HTML-Seite einbauen oder in eine eigene Datei auslagern willst. Doch im Gegensatz zu CSS gibt es für JavaScript keine zwei unterschiedlichen HTML-Tags. 🎎 Für beide Varianten wird stattdessen das <script>-Tag verwendet.

Willst du den JavaScript-Code direkt in die HTML-Datei schreiben, machst du das so. Das <script>-Element darf dabei sowohl im <head> als auch im <body> stehen.

<script> sagHallo = () => { console.log("Hallo, Welt!"); } </script>

Willst du den JavaScript-Code stattdessen auslagern, machst du das so:

<script src="script.js"></script>

Wichtig dabei ist, dass das <script>-Tag sich nicht selbst schließen darf. Du brauchst also zwingend immer auch das schließende </script>-Tag. <script … /> ist nicht erlaubt.

Der Debugger

Reicht dir die Konsole alleine nicht aus, herauszufinden, warum der JavaScript-Code nicht tut, was er soll, kannst du ihn mit dem Debugger untersuchen. 😷 Denn dort siehst du den kompletten Quellcode deiner Seite und kannst mit einem einfachen Klick am linken Rand einen Breakpoint setzen. Lädst du dann die Seite mit F5 neu, bleibt das Programm an genau dieser Stelle stehen. Vorausgesetzt natürlich, es kommt an der Stelle überhaupt vorbei und die Entwicklerwerkzeuge sind geöffnet. Mit den entsprechenden Knöpfen am linken Rand kannst du dann Schritt für Schritt jede Anweisung einzeln ausführen oder das Programm weiterlaufen lassen.

Am rechten Rand siehst du dabei eine Liste aller Variablen, auf die das Programm an der aktuellen Stelle zugreifen kann. Der Aufbau der Liste ist am Anfang zwar etwas gewöhnungsbedürftig, man findet dann aber doch alle Variablen darin, die man sucht. Denn der komische Aufbau kommt daher, dass die Ansicht in verschiedene Bereiche für globale Variablen, lokale Variablen, die Eingangsparameter der aktuellen Funktion und so weiter aufgeteilt ist. 😕

Willst du übrigens in einer größeren Seite mit vielen JavaScript-Dateien nicht erst die Quellcodestelle im Debugger suchen, um dort einen Breakpoint zu setzen, kannst du stattdessen auch einfach die Anweisung debugger in den Quellcode schreiben. Dies bewirkt dasselbe, wie ein mit der Maus gesetzter Breakpoint. Das Programm hält an der entsprechenden Stelle an, wenn die Entwicklerwerkzeuge geöffnet sind.

Wechselst du, während das Programm gerade angehalten wird, in die Konsole, kannst du dort JavaScript-Anweisungen eingegeben, die an der aktuellen Stelle des Programms ausgeführt wird. Das kann einerseits ganz hilfreich sein, um sich einzelne Werte anzusehen, ohne sie in der Liste der Variablen suchen zu müssen. Viel interessanter dabei ist aber, dass du dadurch den Ablauf des Programms ändern kannst, indem du einfach zusätzliche Anweisungen eingibst, die so nicht im Quellcode stehen.

Du siehst: Die Konsole und der Debugger sind zwei unschlagbare Werkzeuge, von denen du regen Gebrauch machen solltest. 👍

  • Übersicht

saveButton.addEventListener("click", …);

Jedes HTML-Element kann eine Reihe von Ereignissen auslösen, auf die man mit JavaScript reagieren kann. Hierfür besitzt jedes Element die Methode addEventListener. Der erste Parameter ist der Name des Ereignisses, der zweite Parameter eine Funktion, die bei Eintreten des Ereignisses aufgerufen wird.

🎓 Merke: Funktionen sind vollwertige Objekte und können daher direkt als Parameter übergeben werden.

🎓 Achtung: Da JavaScript den gesamten Code in einem einzigen Thread ausführt, dürfen die Event-Handler-Funktionen nicht lange laufen. Sonst wird die gesamte Seite ruckelig, weil im selben Thread auch die Bildschirmausgabe erzeugt wird. ⏳

Ereignis Bedeutung
click Einfacher Linksklick mit der Maus oder dem Touchscreen
mouseover Der Mauszeiger befindet sich innerhalb des Elements.
mouseout Der Mauszeiger befindet sich nicht mehr über dem Element.
keydown Beginn eines Tastendrucks auf der Tastatur. Die Taste wurde herunter gedrückt.
keyup Ende eines Tastendrucks auf der Tastatur. Die Taste wurde wieder losgelassen.

Eine gute Übersicht gibt es im Mozilla Developer Network. Da der Browser aber keine komplexen Gesten erkenn kann (Swipe Left, Double Tap, …) muss man hierfür externe Bibliotheken wie Hammer.JS verwenden.

saveButton.addEventListener("click", event => …);

Jeder Event Listener bekommt als ersten Parameter ein Event-Objekt übergeben. In event.target findet sich dann zum Beispiel das HTML-Element, das das Ereignis ausgelöst hat. Es gibt aber noch weitere, interessante Methoden, die gelegentlich nützlich sein können.

window.addEventListener("load", …)

Ein besonders häufig genutztes Ereignis ist das load-Ereignis des window-Objekts. Es zeigt an, dass die HTML-Seite komplett geladen wurde und nun auf das Document Object Model zugegriffen werden kann. Denn der Quellcode aller <script>-Elemente wird sofort ausgeführt, sobald der Browser sie findet. In der Regel ist die Seite zu diesem Zeitpunkt aber noch gar nicht komplett geladen und Zugriffe auf das Document Object Model schlagen fehl.¹

Will man daher beim Laden der Seite zuverlässig auf das Document Object Model zugreifen, muss man erst das load-Ereignis abwarten und in dessen Event Handler alle weiteren Schritte ausführen.

¹ Man nennt das eine „Race Condition”, da der JavaScript-Code mit dem Browser um die Wette läuft. 🏃🏃

Der JavaScript-Code

Das eigentlich Interessante ist der JavaScript-Code. Denn der besteht nicht, wie du es vielleicht erwartet hättest, aus ein paar Funktionen und globalen Variablen, sondern eigentlich nur aus einer großen Funktion, die gleich zu Anfang an das load-Ereignis des window-Objekts gebunden wird. Davor und danach passiert nichts:

window.addEventListener("load", () => { // Hier spielt die Musik … });

Dadurch wird sichergestellt, dass der in der Funktion enthaltene Quellcode erst losläuft, wenn die Seite vollständig geladen wurde und alle HTML-Elemente im Document Object Model (dem DOM-Baum) zur Verfügung stehen. Andernfalls gäbe es eine Race Condition 🏎️, da der JavaScript-Code sofort losläuft, sobald der Browser das <script>-Tag entdeckt. Alle HTML-Elemente nach dem <script>-Tag hat der Browser zu diesem Zeitpunkt aber noch nicht gesehen und daher auch noch nicht in den DOM-Baum eingefügt. Das load-Ereignis wird daher erst ausgelöst, wenn alle Elemete im DOM-Baum enthalten sind 🌴.

Am Anfang der Funktion werden daher erst einmal alle HTML-Elemente ermittelt, die für den weiteren Ablauf benötigt werden. Dies passiert hier einmalig am Anfang, da sich die Elemente im weiteren Verlauf ja nicht mehr ändern und es daher Zeitverschwendung wäre, sie später immer wieder aufs Neue auszulesen.¹ Die HTML-Objekte werden hierfür einfach in verschiedene, lokale Variablen geschrieben:

let buttonPlusOne = document.getElementById("button-plus-one"); let buttonMinusOne = document.getElementById("button-minus-one"); …

¹ In der Praxis macht sich das allerdings kaum bemerkbar. So gesehen ist der wahre Vorteil, dass der Quellcode dadurch ein wenig übersichtlicher wird.

Als nächstes folgen ein paar weitere Variablen mit dem Zustand der Stoppuhr:

  • counter: Die aktuelle Sekundenzahl
  • running: Flag, ob die Uhr gerade läuft
  • lastUpdate: Zeitpunkt (in Millisekunden seit dem 01.01.1970, 0:00 Uhr), wann die Anzeige zuletzt aktualisiert wurde. Diese Variable wird benötigt, um zu erkennen, wann eine weitere Sekunde vergangen ist.

Als nächstes kommen drei Event Handler für die drei Toolbar-Buttons. Zum Beispiel:

buttonPlusOne.addEventListener("click", () => { counter++; });

Das Besondere hieran ist, dass innerhalb der Event-Handler-Funktionen die Variablen der äußeren Funktion direkt verwendet werden können. Mit counter++ wird daher tatsächlich die Sekundenzahl der Stoppuhr hochgezählt, obwohl es sich bei counter eigentlich um eine lokale Variable der äußeren Funktion handelt, die der reinen Lehre nach eigentlich gar nicht mehr existieren dürfte, wenn der Button angeklickt wird. 🤔 Denn eigentlich verschwinden lokale Variablen ja wieder, sobald die Funktion, in der sie definiert wurden, zu Ende gelaufen ist. Dass das hier nicht so ist, liegt daran, dass JavaScript Closures unterstützt und deshalb erkennt, dass die Variablen auch nach dem Ende der load-Funktion noch benötigt werden. Dies ist einer der Hauptunterschiede zu Java und macht viele Dinge möglich, die in Java wesentlich umständlicher sind.

Zum Schluss kommt dann die Funktion updateDisplay, die dafür sorgt, dass die Anzeige regelmäßig aktualisiert wird. Sie wird durch folgenden Kunstgriff immer wieder aufgerufen:

let updateDisplay = () => { … window.requestAnimationFrame(updateDisplay); } window.requestAnimationFrame(updateDisplay);

Diesen Trick solltest du dir merken, wenn du zum Beispiel Spiele in JavaScript entwickeln willst oder wenn du Animationen programmieren willst, für die CSS alleine nicht ausreicht. Was hier passiert ist folgendes:

Der Browser hat intern eine feste Bildrate, wie oft die Anzeige pro Sekunde aktualisiert wird. Alle Anzeigeänderungen und alle Animationen folgen dieser magischen Geschwindigkeit. 🎩 Mit window.requestAnimationFrame(…); kann daher eine JavaScript-Funktion eingeplant werden, die vor dem nächsten Bildaufbau aufgerufen wird. Dadurch wird die Funktion genau einmal ausgeführt. Da unsere Stoppuhr aber permanent laufen muss, plant sich die Funktion einfach immer wieder selbst ein. Dadurch wird sichergestellt, dass JavaScript und das Rendering immer synchron laufen und dass gleichzeitig der Hauptthread des Browsers nicht blockiert.

Der Rest der updateDisplay-Funktion ist eigentlich ziemlich einfach. Im oberen Teil wird einfach die aktuelle Zeit mit der Zeit des letzten Updates verglichen (Variable lastUpdate) und immer, wenn eine Sekunde vorüber ist, die Stoppuhr eins runter gezählt. Ist die Stoppuhr bei Null angekommen, bleibt sie stehen und das Flag running wird wieder auf false gesetzt.

let now = Date.now(); if (now - lastUpdate >= 1000) { lastUpdate = now; if (running && counter > 0) { counter--; } else { running = false; } }

Danach folgen dann ein paar Zeilen, die einfach eine kleine Statusmeldung sowie die aktuelle Sekundenzahl anzeigen. Hierfür wird das Attribut textContent der HTML-Elemente überschrieben, dessen Funktion auf der nächsten Folie nochmal genauer erklärt wird.

counterElement.textContent = counter; if (running) { statusElement.textContent = "Uhr läuft …"; } else { statusElement.textContent = "Gestoppt"; }

Wie der Mensch, so der Hund

Der Haushund (Canis lupus familiaris) ist ein Haustier und wird als Heim- und Nutztier gehalten. Seine wilde Stammform ist der Wolf, dem er als Unterart zugeordnet wird. Wann die Domestizierung stattfand, ist umstritten; wissenschaftliche Schätzungen variieren zwischen 15.000 und 100.000 Jahren vor unserer Zeit.

Im engeren Sinn bezeichnet man als Haushund die Hunde, die überwiegend im Haus gehalten werden, und kennzeichnet damit also eine Haltungsform. Historisch wurde ein Hund, der zur Bewachung des Hauses gehalten wird, als Haushund bezeichnet. Eine weitere Verwendung des Begriffs ist die Einschränkung auf sozialisierte (Haus-)Hunde, also Hunde, die an das Zusammenleben mit Menschen in der menschlichen Gesellschaft gewöhnt und an dieses angepasst sind. Damit wird der Haushund abgegrenzt gegen wild lebende, verwilderte oder streunende Hunde, die zwar auch domestiziert, aber nicht sozialisiert sind.

Der Dingo ist ebenfalls ein Haushund, wird jedoch provisorisch als eigenständige Unterart des Wolfes geführt.

Weitere Informationen

Einige Hunderassen

Zu den kleinsten anerkannten Hunderassen gehört der Chihuahua (FCI-Nr. 218) mit einem Gewicht von 0,5 bis 3,0 kg und einer Widerristhöhe von unter 20 cm; zu den größten Hunderassen zählen die Deutsche Dogge (FCI-Nr. 235) mit einer Widerristhöhe von mindestens 80 cm bei Rüden und der Irish Wolfhound (FCI-Nr. 160) mit bis zu 95 cm. Zu den seltenen Rassen zählt der Curly Coated Retriever oder auch der aus chinesischer Abstammung hervorgehende und seit mehr als 2000 Jahren dokumentierte Shar Pei.

Die richtige Ernährung ist wichtig

Hunde sind, wie Wölfe, in der Lage, ihre Ernährung in Grenzen an das Nahrungsangebot anzupassen. Schon Wölfe ernähren sich nicht ausschließlich von Beutetieren (wobei diese, weitgehend vollständig gefressen, bereits pflanzliche Nährstoffe enthalten), sondern fressen – je nach Futterangebot – auch pflanzliche Nahrung wie Wurzeln, Blätter, Gräser oder Früchte. Im Verlaufe seines Zusammenlebens mit dem Menschen hat sich der Hund zunehmend an dessen Ernährung angepasst und wurde zum Allesfresser. Fleisch als alleiniges Futtermittel für Hunde ist daher unangemessen.

Eine vollwertige Ernährung der Hunde erfolgt am einfachsten mit qualitativ hochwertigem (industriell gefertigtem) Hundefutter. Diese Fütterung versorgt die Tiere mit allen essentiellen Nahrungsbestandteilen. Manche Hundehalter praktizieren eine Hundeernährung mit spezieller Frischkost (BARF). Ernährungsphysiologisch fragwürdig ist die Ernährung mit Speiseresten, da sie Mangelzustände bewirken kann.

Viele menschliche Nahrungs- und Genussmittel sind für Hunde mehr oder wenig giftig, so zum Beispiel Schokolade aufgrund des enthaltenen Theobromins, aber auch Speisezwiebeln, Weintrauben und Rosinen.

In Wirklichkeit handelt es sich also um eine einfache Liste, die durch die beiden Klassen tab-items für die Liste selbst und tab-item für die Einträge drastisch umgestaltet wird. Zusätzlich enthält jeder Eintrag noch ein <div">, mit dem eine kleine Verzierung unterhalb der Beschriftung dargestellt wird.

Die aktive Lasche ist zusätzlich noch mit der Klasse active gekennzeichnet, damit sie sich optisch von den anderen unterscheidet. Eine Aufgabe des JavaScript-Codes wird es daher sein, der aktiven Lasche diese Klasse zuzuordnen und sicherzustellen, dass die anderen Laschen die Klasse nicht besitzen.

Über das data-tab-content-Attribut sind die Listeneinträge mit den HTML-Elementen verknüpft, deren Inhalte ein- oder ausgeblendet werden sollen. #tabpage-1 heißt dabei, dass das Element mit der ID tabpage-1 immer dann sichtbar sein soll, wenn die dazugehörige Lasche aktiv ist. Alle anderen Elemente sollen dann unsichtbar sein. Die zweite Aufgabe des JavaScript-Codes ist daher, die hier angegebenen Elemente sichtbar oder unsichtbar zu schalten.

Weiter unten im HTML-Code befinden sich hierfür deshalb drei <div>-Elemente:

<div id="tabpage-1" class="tab-page"> … </div> <div id="tabpage-2" class="tab-page"> … </div> <div id="tabpage-3" class="tab-page"> … </div>

Sie besitzen jeweils die Klasse tab-page, um auch hier nochmal zu kennzeichnen, dass es sich um die Inhalte zu unseren Tabreitern handelt. Außerdem sorgt das Stylesheet dafür, dass alle Elemente mit dieser Klasse unsichtbar sind. Alles, was der JavaScript-Code daher tun muss, um eine Seite anzuzeigen, ist die Klasse tab-page zu entfernen. Später muss dann einfach die Klasse wieder zugeordnet werden, um die Seite auszublenden.

Der JavaScript-Code

Eine billige Lösung, um die Inhalte bei Klick auf einen Reiter zu wechseln, wäre die Verwendung des onclick-Attributs im HTML, ungefähr so:

<ul class="tab-items"> <li class="tab-item active" data-tab-content="#tabpage-1" onclick="switchTabPage(event.target)" > Über den Haushund <div class="tab-indicator"></div> </li> … </ul>

Man bräuchte dann nur noch eine switchTabPage()-Funktion definieren, in der alle notwendigen Anpassungen am DOM vorgenommen werden. Jedoch wäre das keine sonderlich schöne Lösung, da der HTML-Code dadurch nur unnötig unübersichtlich und unsemantisch wird. Viel besser ist es daher, die Tabreiter beim Laden der Seite automatisch zu erkennen und mit Event Handlern zu versehen. Aus diesem Grund beginnt auch dieser JavaScript-Code wieder mit dem gut alten load-Event des window-Objekts:

window.addEventListener("load", () => { // Hier kümmern wir uns um alles … });

Innerhalb der Load-Funktion passiert dann folgendes:

  1. Am Anfang steht die innere Funktion switchTabPage(), mit der die sichtbaren Inhalte umgeschaltet werden.
  2. Dann werden alle Tablaschen im HTML-Dokument gesucht und in der Variablen tabItems abgelegt.
  3. Zum Schluss geht das Programm über alle gefundenen Laschen drüber und prüft für jede, ob sie aktiv ist und daher ihr Inhalt angezeigt werden soll. Zusätzlich wird jede Lasche mit einem Event Handler für das click-Ereignis versehen.

Insgesamt hat die Load-Funktion daher folgenden, groben Aufbau:

// Ausgehend vom angeklickten <li>-Element suche alle // anderen <li>-Elemente, indem du die Kinder seines // Elternelements untersuchst clickedTabItem.parentNode.childNodes.forEach(tabItem => { // Nur weiter machen, wenn des gefundene Element ein // HTML-Element ist. Somit kann der Anwender auf fehlende oder falsche Eingaben aufmerksam gemacht werden, bevor sie überhaupt versendet werden. 🔔

Der HTML-Code

Formulare haben wir bisher noch nicht kennengelernt. Deshalb erfolgt hier eine kurze Erklärung, was die einzelnen HTML-Elemente bedeuten.

<form action="…" method="…">

Ein Formular fasst ein oder mehrere Eingabefelder zusammen, die zur Weiterverarbeitung an den Server geschickt werden können. Hierfür sendet der Browser eine HTTP-Anfrage an die im action-Attribut hinterlegte URL. method kann dabei entweder GET oder POST sein und legt fest, auf welche Art die Daten übertragen werden. Details dazu erfährst du, sobald wir uns mit HTTP auseinandersetzen.

<label>

Hiermit werden die Feldbezeichner definiert. Durch das for-Attribut wird dabei festgelegt, auf welches Feld sich der Bezeichner bezieht. Dies macht zwar keinen Unterschied in der Darstellung, ist aber für Screen Reader und andere Ausgabegeräte eine wichtige Information.

<input>

Dieses Element definiert, wie der Name schon sagt, ein Eingabefeld. Das type-Attribut legt dabei fest, welche Daten das Feld versteht (text, email, phone, und so weiter).

<select>

Hiermit wird eine Dropdown-Liste mit vorgegebenen Auswahlmöglichkeiten definiert.

<textarea>

Dies ist ein mehrzeiliges Eingabefeld, in das der Anwender beliebig langen Text schreiben kann. Von Haus aus wird allerdings nur unformatierter Text unterstützt. Willst du stattdessen auch formatierte Eingaben erlauben, benötigst du eine zusätzliche Bibliothek wie zum Beispiel TineMCE oder CKEditor.

<input type="submit">

Das ist der Knopf, den der Anwender drücken muss, um das Formular an den Server zu schicken.

Achtung: Dadurch wird automatisch eine neue Seite vom Server angefordert. Willst du nur die Daten an den Server schicken, ohne die Seite zu verlassen, musst du noch ein bisschen warten, bis wir uns AJAX anschauen.

Der JavaScript-Code

Die Verbindung zum JavaScript-Code erfolgt über das submit-Event, das das Formular wirft, sobald der Anwender es verschicken möchte. Hier haben wir es uns einfach gemacht, und rufen über das onsubmit-Attribut im HTML die JavaScript-Funktion validateForm() auf.

Das zu überprüfende Formular steht in der Variablen event.target. Dieses Objekt hat für jedes Eingabefeld ein gleichnamiges Attribut, dessen Name sich aus dem name-Attribut im HTML ergibt. Und dieses besitzt widerum ein value-Attribut für seinen Inhalt:

let validateForm = event => { let form = event.target; … // Vorname muss vorhanden sein if (form.firstname.value == "") { … } … }

Der Rest ist ziemlich einfach. Siehst du dir die Funktion etwas genauer an, wirst du sehen, dass hier einfach nur ein paar if-Abfragen hintereinander stehen, um die Inhalte der Formularfelder zu überprüfen. Allerdings muss der Browser am Ende natürlich wissen, ob die Prüfung erfolgreich war oder nicht. Standardmäßig nimmt er dabei an, dass alles in Ordnung ist und das Formular abgeschickt werden kann. Um dies zu verhindern, wird am Ende der Funktion folgende Methode aufgerufen:

//if (!okay) { event.preventDefault(); //}

Anhand der Kommentare siehst du, dass dies eigentlich nur erfolgen sollte, wenn die Prüfung des Formulars fehlgeschlagen ist. Da wir hier aber nicht wirklich einen Server im Hintergrund haben, der mit den Eingaben dieses Beispiels etwas anfangen kann, wird die Methode hier immer aufgerufen.
